# Problem-Solution Structure

## Overview

The simplest and most common ad structure. Show problem, introduce solution, demonstrate result.

---

## Structure

### Part 1: Problem (30-40%)
Show the pain point or need

### Part 2: Solution (30-40%)
Introduce product as answer

### Part 3: Result (20-30%)
Show happy outcome

---

## For 15-Second Ad

```yaml
Problem (5s): Character struggling
Solution (5s): Product introduced and used
Result (5s): Happy character + logo
```

---

## For 30-Second Ad

```yaml
Problem (10s): 
  - Establish pain point (5s)
  - Show frustration (5s)

Solution (12s):
  - Introduce product (4s)
  - Show features (4s)
  - Demonstrate use (4s)

Result (8s):
  - Happy outcome (5s)
  - CTA + logo (3s)
```

---

## Variations

### 1. Before/After
```yaml
Before (40%): Life without product
After (40%): Life with product
CTA (20%): Logo and call to action
```

### 2. Problem-Agitate-Solve
```yaml
Problem (25%): Show issue
Agitate (25%): Make it worse, emphasize pain
Solve (35%): Product fixes everything
CTA (15%): Logo
```

### 3. Multiple Problems
```yaml
Problem_1 (20%): First pain point
Problem_2 (20%): Second pain point
Solution (40%): Product solves both
Result (20%): Happy ending
```

---

## Visual Techniques

### Problem Section
```yaml
color_grading: Desaturated, cool tones
lighting: Harsh or dim
music: Tense, uncomfortable
pacing: Slow or chaotic
```

### Solution Section
```yaml
color_grading: Neutral, transitioning to warm
lighting: Improving
music: Hopeful, building
pacing: Steady, confident
```

### Result Section
```yaml
color_grading: Saturated, warm tones
lighting: Bright, pleasant
music: Uplifting, resolved
pacing: Smooth, satisfied
```

## Examples

### Example 1: Cleaning Product (15s)
```yaml
S1 (5s): Dirty kitchen, frustrated person
S2 (5s): Person uses cleaning product, wipes counter
S3 (5s): Sparkling clean kitchen, happy person, logo
```

### Example 2: App (30s)
```yaml
S1 (5s): Person overwhelmed with tasks, messy desk
S2 (5s): Multiple notifications, stress building
S3 (4s): Discovers app on phone
S4 (4s): App interface showing organization
S5 (4s): Person using app, tasks getting done
S6 (5s): Clean desk, calm person, productive
S7 (3s): App logo + "Get organized today"
```

### Example 3: Food Product (30s)
```yaml
S1 (6s): Family dinner time, everyone hungry and tired
S2 (4s): Parent stressed about cooking
S3 (5s): Product introduced from freezer
S4 (5s): Quick preparation shown
S5 (5s): Family enjoying meal together, smiling
S6 (5s): Product shot + logo + "Dinner solved"
```

## When to Use

**Best for**:
- Clear pain points
- Functional products
- Direct response ads
- B2B products
- Problem-solving services
- Household products
- Apps and software

**Universal appeal**:
- Easy to understand
- Works for any audience
- Proven effective
- Simple to produce

---

## Tips for Effectiveness

### 1. Make Problem Relatable
```yaml
approach: Show common, recognizable frustration
avoid: Exaggerating to point of unrealistic
```

### 2. Show Clear Causation
```yaml
approach: Product directly solves shown problem
avoid: Vague or unrelated benefits
```

### 3. Demonstrate, Don't Tell
```yaml
approach: Show product working
avoid: Just talking about features
```

### 4. Emotional Payoff
```yaml
approach: Show relief, happiness, satisfaction
avoid: Just showing product
```

---

## Characteristics

- Linear, chronological
- Clear cause and effect
- Easy to follow
- Universal structure
- Proven effectiveness
- Works for any product
- Simple to produce
- Audience expects it
